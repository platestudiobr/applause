🎮 Fluxograma Interativo - Gamificação Applause + Plate Visual

Fluxo 1: Trigger por Chat
1
Usuário completa meta/missão
Vendedor atinge alvo (ex: R$ 1.000 em vendas)
2
Insere dados no chat
Texto: "Completei a missão de R$ 1.000"
TRIGGER: Detecção de texto/comando
3
Sistema detecta padrão
Backend identifica keyword/comando que ativa animação
AÇÃO: Envio de signal ao frontend
4
Animação dispara
Personagem reage com celebração, confetes, feedback visual
RESULTADO: Sucesso animado
5
Próxima interação
Mistura: animação + mensagem do chat com próxima missão
CONTEXTO: Chat continua fluxo normal
Fluxo 2: Trigger por Clique
1
Elemento interativo aparece
Caixa surpresa, baú, presente animado no canvas
2
Convite visual para interagir
Mensagem no chat: "Clique na caixa para ver sua próxima missão"
TRIGGER: Click direto no elemento
3
Usuário clica no elemento
Interação direta com a animação na tela
AÇÃO: Event listener dispara callback
4
Animação responde
Caixa abre, confetes explodem, elemento se transforma
RESULTADO: Feedback visual + surpresa
5
Conteúdo revelado
Próxima missão, prêmio, pontos aparcem no chat ou na tela
CONTEXTO: Continua interação híbrida

Tipos de Gatilhos & Reações

💬
Chat Input
Usuário escreve no chat, palavra-chave ativa animação no canvas
🖱️
Click Direto
Usuário clica em elemento animado, dispara sequência de frames
⏱️
Time-Based
Animação dispara automaticamente após X segundos ou ação anterior
🎯
Meta Atingida
Sistema detecta que usuário alcançou milestone, comemora automaticamente
🔔
Notificação Visual
Personagem chama atenção do usuário com animação leve e contínua
🎬
Sequência Encadeada
Uma animação termina, automáticamente a próxima começa (storytelling)
Arquitetura de Implementação
Frontend (Plate Studio)
Desenvolvimento das animações em Lottie / Three.js / WebGL. Componentes React para o canvas de animação. Estrutura pronta para receber triggers externos.
Backend (Applause)
Lógica de detecção de triggers (chat input, click events). Sistema de emissão de eventos para o frontend. Controle de fluxo, pausas, sequências de animações.
API de Comunicação
WebSocket ou HTTP para enviar sinais trigger → frontend. Payload simples com ID da animação, parâmetros, timing. Feedback do usuário volta para o backend (analytics).
Fluxo de Estados
Repouso → Aguardando trigger → Em execução → Pausa → Próxima. Cada estado pode ser interrompido ou encadeado. Memória de animações já vistas (não repetir).
Exemplo Prático
Usuário: "Vendi R$ 1000" → Chat detecta → Backend envia { type: 'celebrate', icon: 'trophy', particles: 'confetti' } → Animação executa → Chat mostra próxima meta.
Vantagem da Proposta
Gamificação deixa de ser abstrata. Usuário vê feedback imediato e visceral. Aumenta engajamento, retenção, competição saudável entre vendedores.
